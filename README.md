# Пародия на банковскую систему

Данный проект представляет собой поэтапную реализацию банковской системы.

## Общая идея
Система предназначена для управления пользователями, их счетами и финансовыми операциями. Проект на основе REST API, с безопасностью и асинхронным взаимодействием сервисов.

## Краткое описание

### Базовая банковская система (lab1)
- Реализация доменной модели: пользователь, счёт, операции.
- Поддержка создания пользователей и счетов, управления друзьями, переводов и комиссий.
- Покрытие бизнес‑логики unit‑тестами (JUnit, моки репозиториев).
- Автоматическая генерация JavaDoc.
- Настроен CI (GitHub Actions) с использованием Maven или Gradle.
- Проект разделён на три модуля.

### DataAccess layer  
- Хранение данных в PostgreSQL.
- Использование Hibernate/JPA для Data Access слоя.
- Миграции базы данных с помощью Flyway.

### Presentaation layer with REST API и Spring Boot
- Перевод приложения на Spring Boot и Spring Framework.
- Реализация REST API для всех сценариев
- Использование JpaRepository.
- Фильтрация пользователей, счетов и операций.
- DTO
- DI
- Swagger‑документация для всех эндпоинтов.
- Строгое соблюдение REST‑подхода

### API Gateway
- Реализация отдельного приложения API Gateway c использование SpringSecurity
- Аутентификация и авторизация всех HTTP‑запросов.
- Разделение ролей: Admin и Client.
- Закрытая регистрация пользователей (только администратор).
- Контроль доступа к данным и операциям.
- Шифрование паролей.
- Взаимодействие Gateway с Bank‑service исключительно через HTTP.

### Storage
- Реализация отдельного сервиса
- Использование Kafka или RabbitMQ для получения информации
- Публикация событий о создании и изменении пользователей и счетов.
- Разделение топиков: client‑topic и account‑topic.
- Хранение событий в собственной базе данных Storage‑сервиса.
- Полная изоляция Gateway от Storage.


## Итоговая архитектура
- **Bank‑service** — основная бизнес‑логика и REST API.
- **Gateway** — безопасность и контроль доступа.
- **Storage** — асинхронное хранение событий.
